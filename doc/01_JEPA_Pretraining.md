# Step 1: JEPA Pre-training

The first phase of Guacarpov's pipeline (`JEPA/train.py`) is purely self-supervised. We don't train the model to predict what move a Grandmaster played; instead, we train it to understand the fundamental physics and strategy of the board.

## The Joint-Embedding Predictive Architecture (JEPA)

Instead of predicting exact pixel reconstruction (like an Autoencoder) or exact piece locations, JEPA predicts the *latent representation* of the future. 

1.  **Context Encoder (Online):** Processes the board state at time `t` into a continuous latent vector.
2.  **Target Encoder (Frozen/EMA):** Processes the board state at time `t+1`. Its weights are not updated via backpropagation, but rather by an Exponential Moving Average of the Context Encoder. This provides a stable, slowly evolving target.
3.  **Predictor:** A small neural network that takes the Context Latent and tries to output the Target Latent. 

Because the predictor is not told *what* move was played between `t` and `t+1`, it is forced to learn general strategic dynamics and probable continuations to minimize its prediction error.

## The Loss Functions

During pre-training, Guacarpov balances three distinct losses:

### 1. JEPA Prediction Loss (MSE)
The Mean Squared Error between the predictor's output and the frozen target encoder's output. This is the core engine of learning.

### 2. Value Grounding (Huber Loss)
To ensure the latent representations capture actual "chess strategy" and not just piece geometry, an auxiliary value head predicts the final game outcome (-1, 0, +1) directly from the context latent. This physically forces the representations to become linearly separable by positional advantage.

### 3. Variance Regularization (VICReg Style)
A major risk in JEPA architectures is **Representation Collapse** — where the model finds a lazy shortcut by outputting the exact same constant vector for every single board state (resulting in a prediction MSE of 0).

*   **The Fix:** We calculate the standard deviation of the features across the batch. If the standard deviation drops below 1.0, we apply a hinge loss to push it back up.
*   **Design Choice:** Crucially, this variance loss is applied to the *Context Latents* and the *Predicted Latents*, because these are generated by the active network (which has gradients). Applying it to the frozen Target Latents would result in zero gradients and fail to prevent collapse.
